# Summary Report: “AWS Cloud Mastery Series #3 – Security Pillar”

### General Introduction

**AWS Cloud Mastery Series #3 – Security Pillar** was a **morning workshop** held on **November 29, 2025** at the **AWS Vietnam Office, Bitexco Financial Tower, Ho Chi Minh City**.  
The session focused on the **AWS Well-Architected Security Pillar**, providing a deep dive into security best practices, identity management, threat detection, infrastructure protection, data security, and incident response.  
The workshop was designed for **developers, solution architects, and cloud practitioners** looking to enhance their skills in designing **secure and resilient AWS workloads**.

---

### Event Objectives

- Understand the **principles of the AWS Well-Architected Security Pillar**.  
- Learn best practices for **identity and access management, monitoring, and infrastructure protection**.  
- Explore **data encryption, secrets management, and incident response automation**.  
- Gain insights into **common security pitfalls** and real-world scenarios in Vietnam.  
- Create a roadmap for **advanced security learning**, including **AWS Security Specialty** and **Solutions Architect Professional**.

---

### Event Schedule & Key Sessions

#### 8:30 – 8:50 AM | Opening & Security Foundation
- Role of the **Security Pillar** within the Well-Architected Framework.  
- Core principles: *Least Privilege*, *Zero Trust*, *Defense in Depth*.  
- **Shared Responsibility Model** explained.  
- Overview of **top cloud threats in Vietnam**.

#### 8:50 – 9:30 AM | Pillar 1 – Identity & Access Management (IAM)
- Modern IAM architecture: Users, Roles, Policies, avoiding long-term credentials.  
- IAM Identity Center: SSO and permission sets.  
- Service Control Policies (SCPs) & permission boundaries for multi-account setups.  
- MFA, credential rotation, and access analysis.  
- Mini demo: Validate IAM policies and simulate access scenarios.

#### 9:30 – 9:55 AM | Pillar 2 – Detection & Continuous Monitoring
- Continuous monitoring with **CloudTrail, GuardDuty, and Security Hub**.  
- Logging at all levels: VPC Flow Logs, ALB logs, S3 logs.  
- Alerting and automation using **EventBridge**.  
- Detection-as-Code: Infrastructure and rules automation.

#### 9:55 – 10:10 AM | Coffee Break

#### 10:10 – 10:40 AM | Pillar 3 – Infrastructure Protection
- Network & workload security: VPC segmentation, private vs public placement.  
- Security Groups vs NACLs: best practice models.  
- AWS WAF, Shield, and Network Firewall.  
- Basic workload protection for EC2, ECS, and EKS.

#### 10:40 – 11:10 AM | Pillar 4 – Data Protection
- Key management with **KMS**: policies, grants, rotation.  
- Encryption at-rest & in-transit for S3, EBS, RDS, DynamoDB.  
- Secrets Manager & Parameter Store patterns for rotation.  
- Data classification and access guardrails.

#### 11:10 – 11:40 AM | Pillar 5 – Incident Response (IR)
- AWS incident response lifecycle.  
- IR playbook examples:
  - Compromised IAM key  
  - S3 public exposure  
  - EC2 malware detection  
- Snapshotting, isolation, evidence collection.  
- Automated responses with **Lambda** and **Step Functions**.

#### 11:40 – 12:00 PM | Wrap-Up & Q&A
- Review of the 5 security pillars.  
- Discussion of common pitfalls and real-world security practices in Vietnam.  
- Guidance on **security learning roadmap** (Security Specialty, SA Pro).
